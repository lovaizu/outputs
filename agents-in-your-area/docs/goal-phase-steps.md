# Goal Phase — Step List v3

Produced by integrating two independent expert reviews (product discovery + AI agent design)
and a subsequent re-review pass. All High, Medium, and Low findings addressed.

---

## Step List

| ID | Name | Type | Purpose | CCS In (key components) | CCS Out (key components, success path) | Notes |
|----|------|------|---------|------------------------|---------------------------------------|-------|
| G0 | Goal Intake | [I] | Initialize CCS from the /hi hearing transcript so that all downstream steps start from a verified common ground. | episodic_trace (raw /hi transcript) | episodic_trace (hearing summary), semantic_gist (initial framing), focal_entities (named actors and objects from hearing), goal_orientation (stated user goal), uncertainty_signal (gaps and ambiguities in hearing) | If /hi transcript is absent, set uncertainty_signal (gap: no hearing transcript) and halt with a request to provide input. |
| G1 | User Identification | [E] | Confirm which user segment is the target of the goal so that subsequent elicitation is scoped to a real, addressable population. | episodic_trace (G0 hearing summary), semantic_gist (initial framing), goal_orientation, uncertainty_signal | episodic_trace (confirmed segment), focal_entities (target segment, named stakeholders), goal_orientation (refined), uncertainty_signal (remaining unknowns) | |
| G2 | Elicitation Scoping — Situation | [I] | Define the exact questions to ask in G3 and the termination condition that will signal a satisfactory Situation statement. | episodic_trace (G1 output), focal_entities (target segment), goal_orientation, uncertainty_signal | semantic_gist (question set), goal_orientation, predictive_cue (termination condition: Situation passes factual specificity and scope tests) | Termination condition must be a checkable predicate, not a judgment. |
| G3 | Situation Draft | [E] | Elicit context from the expert and propose a first Situation statement scoped to the confirmed user segment. | episodic_trace (G1 output), semantic_gist (G2 question set), focal_entities, goal_orientation, predictive_cue (termination condition), uncertainty_signal | episodic_trace (expert responses), semantic_gist (Situation draft v1), focal_entities (updated), goal_orientation, uncertainty_signal (open factual gaps) | |
| G4 | Situation Revision ×N | [E] | Revise the Situation statement per /gm feedback until it meets the G2 termination condition or iteration limit is reached. | episodic_trace (prior exchange), semantic_gist (current Situation draft), focal_entities, goal_orientation, predictive_cue (termination condition), uncertainty_signal | episodic_trace (updated), semantic_gist (revised Situation draft), focal_entities, goal_orientation, uncertainty_signal | **Circuit-breaker:** Max N=5. On N=5 without /ty, fire an escalation Turn presenting a divergence summary and requesting explicit expert direction before continuing. |
| G5a | Situation Evidence Research | [R] | Gather qualitative evidence (market data, user research, analyst reports) that corroborates or challenges the current Situation draft. | semantic_gist (Situation draft), focal_entities, goal_orientation, uncertainty_signal | retrieved_artifacts (qualitative evidence, each tagged with source and relevance), uncertainty_signal | **Parallel with G5b** (‖). Results merged in G5c. **Failure mode:** If research yields no usable data, set uncertainty_signal (gap: qualitative evidence unavailable — [reason]) and complete; G5c surfaces this to G6. |
| G5b | Situation Domain Quantitative Research | [R] | Gather quantitative data (size, frequency, cost, adoption metrics) scoped strictly to the Situation domain — not to any candidate Pain. | semantic_gist (Situation draft), focal_entities, goal_orientation, uncertainty_signal | retrieved_artifacts (quantitative data, each tagged with source, metric, and Situation relevance), uncertainty_signal | **Parallel with G5a** (‖). **Scope constraint:** Prohibited from pre-selecting or implying Pain candidates. Outputs inform Pain elicitation without framing it. **Failure mode:** Same pattern as G5a. |
| G5c | Research Merge | [I] | Merge retrieved_artifacts from G5a and G5b into a single coherent set so that replacement semantics does not drop either branch's output. | retrieved_artifacts (G5a output), retrieved_artifacts (G5b output), uncertainty_signal (from each branch) | retrieved_artifacts (merged, deduplicated, branch tags preserved), uncertainty_signal (consolidated: gaps from either branch), semantic_gist (evidence summary) | Required because replacement semantics means one branch would overwrite the other without this merge Turn. G5c is the authoritative evidence source for G6 onward. |
| G6 | Situation Validation | [I] | Validate the Situation draft against merged evidence; flag any factual mismatch, unsupported claim, or critical evidentiary gap. | semantic_gist (Situation draft), retrieved_artifacts (G5c), goal_orientation, uncertainty_signal | semantic_gist (validation findings: supported / unsupported claims, flagged gaps), goal_orientation, uncertainty_signal (flag raised if mismatch), predictive_cue (PASS or BLOCK with reason) | G6 raises flags only; G6b presents them to the expert. |
| G6b | Situation Validation Review | [E] | Present G6's validation findings and obtain an explicit expert decision to proceed or revise. | semantic_gist (G6 validation findings), goal_orientation, uncertainty_signal (G6 flag), predictive_cue (PASS or BLOCK) | episodic_trace (expert decision), semantic_gist (updated with resolution), goal_orientation, predictive_cue (re-entry: G3 if BLOCK and expert chooses revision) | If G6 produced no flag (PASS), this step is a pass-through. If BLOCK, expert issues /gm to re-enter G3 or /ty to override with rationale. |
| G7 | Elicitation Scoping — Pain | [I] | Define Pain elicitation questions and termination condition, informed by the validated Situation and merged evidence. | semantic_gist (validated Situation, G5c evidence summary), focal_entities, goal_orientation, uncertainty_signal | semantic_gist (Pain question set), goal_orientation, predictive_cue (termination condition: Pain is an observable user symptom, not a root cause or solution) | |
| G8 | Pain Candidates Draft | [E] | Elicit from the expert and propose multiple candidate Pain statements, each framed as an observable user symptom. | episodic_trace (G6b exchange), semantic_gist (G7 question set, validated Situation), focal_entities, goal_orientation, predictive_cue (G7 termination condition), uncertainty_signal | episodic_trace (expert responses), semantic_gist (candidate Pain list), focal_entities (updated), goal_orientation, uncertainty_signal | |
| G9 | Pain Revision ×N | [E] | Revise the candidate Pain set per /gm feedback until it meets G7's termination condition or iteration limit is reached. | episodic_trace (prior exchange), semantic_gist (current candidate Pain set), focal_entities, goal_orientation, predictive_cue (G7 termination condition), uncertainty_signal | episodic_trace (updated), semantic_gist (revised candidate Pain set), focal_entities, goal_orientation, uncertainty_signal | **Circuit-breaker:** Max N=5. Same escalation pattern as G4. |
| G10 | Pain Prioritization and Selection | [E] | Force selection of exactly one primary Pain from the candidate set, with explicit expert rationale for the choice. | episodic_trace (G9 output), semantic_gist (revised candidate Pain set), focal_entities, goal_orientation, retrieved_artifacts (G5c evidence), uncertainty_signal | episodic_trace (expert selection and rationale), semantic_gist (selected Pain with rationale), focal_entities (updated), goal_orientation, uncertainty_signal | Selection must name one Pain; partial selections are not accepted. Expert issues /ty to confirm or /gm to reopen G9. |
| G11 | Pain Evidence Collection | [R] | Gather quantitative data, user feedback, and analogues specific to the selected Pain, incorporating prior Situation artifacts. | semantic_gist (selected Pain, validated Situation), retrieved_artifacts (G5c as prior context), focal_entities, goal_orientation, uncertainty_signal | retrieved_artifacts (Pain-specific evidence, incorporating prior Situation artifacts; each tagged with source and Pain relevance), uncertainty_signal | This Turn writes a fresh CCS that includes both prior Situation artifacts and new Pain evidence — it does not append in place. **Failure mode:** Set uncertainty_signal (gap: Pain evidence unavailable — [reason]); G12 reads this signal and decides whether to block. |
| G12 | Pain Lock | [I] | Confirm that the selected Pain is an observable user symptom only — not a root cause, not a solution — and that evidence supports its significance. | semantic_gist (selected Pain, G10 rationale), retrieved_artifacts (G11 Pain evidence), focal_entities, goal_orientation, uncertainty_signal | semantic_gist (locked Pain statement), goal_orientation, uncertainty_signal (updated), predictive_cue (LOCKED or BLOCK with reason) | If Pain contains causal language or solution framing, set BLOCK and route back to G10 via predictive_cue. Pain Lock is a prerequisite for Root Cause Analysis (G13). |
| G13 | Root Cause Analysis | [I] | Trace the root cause of the locked Pain to a depth of ≤ 3 causal hops, producing a cause chain that Benefit elicitation can invert. | semantic_gist (locked Pain from G12), retrieved_artifacts (G11 Pain evidence), focal_entities, goal_orientation, uncertainty_signal | semantic_gist (root cause chain: Pain → cause 1 → … → root, max 3 hops), focal_entities (updated with causal actors), goal_orientation, relational_map (causal structure), uncertainty_signal | Root cause must analyze the confirmed, locked Pain — not the candidate set. Positioned after G12 for this reason. If causal chain cannot be established within 3 hops, set uncertainty_signal (gap: causal chain unclear). |
| G14 | Similar Cases Research — Pain Axis | [R] | Gather analogous cases and precedents where the same root cause was addressed, to inform Benefit scoping. | semantic_gist (locked Pain, root cause chain from G13), relational_map (causal structure), focal_entities, goal_orientation, uncertainty_signal | retrieved_artifacts (analogous cases, each tagged with root cause match and Benefit outcome), uncertainty_signal | Precedes Elicitation Scoping — Benefit (G15) so that G15 can be informed by real analogues. **Failure mode:** Set uncertainty_signal (gap: no analogous cases found — [reason]); G15 reads this and adjusts scoping confidence accordingly. |
| G15 | Elicitation Scoping — Benefit | [I] | Define Benefit elicitation questions and termination condition, informed by the root cause chain and G14 analogous cases. | semantic_gist (locked Pain, root cause chain), retrieved_artifacts (G14 analogous cases), focal_entities, goal_orientation, uncertainty_signal (G14 gaps) | semantic_gist (Benefit question set), goal_orientation, predictive_cue (termination condition: Benefit traces to root cause, is not the inverse of Pain, is strategically addressable) | Follows G14 so the question set is grounded in research, not vacuum reasoning. |
| G16 | Benefit Draft | [E] | Propose a Benefit statement derived from the root cause chain; present alongside the inverse-of-Pain test result for expert review. | episodic_trace (G12 Pain Lock, G13 root cause chain), semantic_gist (G15 question set, G14 analogues), focal_entities, goal_orientation, predictive_cue (G15 termination condition), uncertainty_signal | episodic_trace (expert responses), semantic_gist (Benefit draft v1, with inverse-of-Pain test result noted), focal_entities (updated), goal_orientation, uncertainty_signal | Inverse-of-Pain test is presented for transparency; structural validation is deferred to G17b. |
| G17 | Benefit Revision ×N | [E] | Revise the Benefit draft per /gm feedback until it meets G15's termination condition or iteration limit is reached. | episodic_trace (prior exchange), semantic_gist (current Benefit draft), focal_entities, goal_orientation, predictive_cue (G15 termination condition), uncertainty_signal | episodic_trace (updated), semantic_gist (revised Benefit draft), focal_entities, goal_orientation, uncertainty_signal | **Circuit-breaker:** Max N=5. Same escalation pattern as G4. |
| G17b | Benefit Quality Validation | [I] | Independently verify that the Benefit draft (a) is not the inverse of Pain and (b) traces structurally to the root cause from G13, not merely to the Pain surface. | semantic_gist (revised Benefit draft from G17, root cause chain from G13, locked Pain from G12), focal_entities, goal_orientation | semantic_gist (validation findings: root-cause trace result, inverse-of-Pain test result), goal_orientation, uncertainty_signal (PASS or BLOCK with specific structural failure), predictive_cue (PASS → G18; BLOCK → re-enter G17 with failure detail) | Independent structural check — does not rely on G16's self-applied test. If BLOCK, predictive_cue routes back to G17 with the specific failure. G18 only receives PASS cases. |
| G18 | Benefit Strategic Relevance Check | [E] | Confirm with the expert that the validated Benefit is a current strategic priority worth pursuing in the Approach phase. | episodic_trace (G17 exchange), semantic_gist (Benefit draft, G17b validation findings), focal_entities, goal_orientation, uncertainty_signal | episodic_trace (expert confirmation or redirect), semantic_gist (Benefit with strategic relevance confirmed), goal_orientation, uncertainty_signal | A /gm here may signal a need to revisit root cause or Pain selection; G-REDIRECT handles routing. |
| G19 | Benefit Lock + Evidence Compression | [I] | Lock the Benefit statement and drop all artifacts not cited in any Lock step output or locked semantic_gist component. | semantic_gist (confirmed Benefit, all prior Lock step outputs: G6, G12), retrieved_artifacts (full evidence set), focal_entities, goal_orientation | semantic_gist (locked Benefit), retrieved_artifacts (cited artifacts only), focal_entities (pruned), goal_orientation, uncertainty_signal | **Citation definition:** An artifact is "cited" if referenced in the CCS Out of any Lock step (G6, G12, G19) or in any locked semantic_gist component. All uncited artifacts are dropped. |
| G20 | Consistency Validation | [I] | Verify that Situation, Pain, and Benefit form a logically consistent chain; produce a re-entry routing instruction if any inconsistency is found. | semantic_gist (locked Situation, locked Pain, locked Benefit), relational_map (G13 causal structure), focal_entities, goal_orientation | semantic_gist (consistency check findings), goal_orientation, uncertainty_signal (inconsistency detail if found), predictive_cue (re-entry step ID: Situation–Pain mismatch → G3; Pain–Benefit mismatch → G15; multi-element → earliest inconsistent element) | Re-entry routing is required: G20 must produce predictive_cue (re-entry step ID) whenever an inconsistency is found. The flag alone is insufficient. |
| G21 | AS Draft + Out-of-Scope | [E] | Draft Acceptance Scenarios in the required format and define what is explicitly out of scope. | semantic_gist (locked Situation, Pain, Benefit, root cause chain), focal_entities, goal_orientation, relational_map, uncertainty_signal | episodic_trace (expert input), semantic_gist (AS draft set, out-of-scope list), focal_entities, goal_orientation, uncertainty_signal | **AS format:** Each AS states an observable, user-visible state; no solution mechanism; no internal system state. Example: "User can [observable action] without [pain symptom]." G23's format-compliance check references this definition. |
| G22 | AS Revision ×N | [E] | Revise the AS set per /gm feedback until testability requirements are met or iteration limit is reached; on re-entry from G24, consume G23's uncertainty_signal to target specific failures. | episodic_trace (prior exchange), semantic_gist (current AS set, out-of-scope list), focal_entities, goal_orientation, predictive_cue (testability termination condition), uncertainty_signal (G23 per-AS failure detail if re-entering from G24) | episodic_trace (updated), semantic_gist (revised AS set), focal_entities, goal_orientation, uncertainty_signal | **Circuit-breaker:** Max N=5. Same escalation pattern as G4. On re-entry from G24, CCS In must include G23's uncertainty_signal so revisions target the specific flagged items. |
| G23 | AS Testability Check | [I] | Verify each AS against three criteria: (1) falsifiable, (2) Pain-traceable — resolves the Pain, not merely topically related, (3) format-compliant per G21's AS format definition. | semantic_gist (AS set, G21 AS format definition, locked Pain from G12), relational_map, focal_entities, goal_orientation | semantic_gist (per-AS check results: PASS or FAIL with criterion), uncertainty_signal (per-AS failure detail for any FAIL), focal_entities (AS set with flagged items), goal_orientation, predictive_cue (overall: PASS → G24 success path; any FAIL → G24 failure path) | Criterion 2 requires that if all AS are met, the Pain is no longer observable — "topically related" is insufficient. |
| G24 | AS Lock | [I] | Lock the AS set if G23 passes; if G23 flagged failures, emit failure-path CCS that routes back to G22 for targeted revision. | semantic_gist (G23 per-AS check results), uncertainty_signal (G23 per-AS failure detail), focal_entities (flagged AS set), goal_orientation, predictive_cue (G23 overall: PASS or FAIL) | **Success path:** semantic_gist (locked AS set), goal_orientation, predictive_cue (proceed to G25). **Failure path:** uncertainty_signal (per-AS failure detail from G23), focal_entities (AS set with flagged items), predictive_cue (re-entry: G22). | Failure-path CCS Out is distinct from success-path. G22 re-entry must consume the failure-path uncertainty_signal. |
| G25 | Traceability Verification | [I] | Verify the complete AS→Pain→Situation chain: each AS traces to locked Pain, Pain traces to validated Situation, no orphaned elements. | semantic_gist (locked AS set, locked Pain, locked Situation), relational_map (G13 causal structure), focal_entities, goal_orientation | semantic_gist (traceability report: chain status per AS), goal_orientation, uncertainty_signal (broken links if any), predictive_cue (PASS → G26; broken link → re-entry at earliest broken element) | |
| G26 | Gate Preparation | [I] | Package all locked artifacts, the traceability report, and the evidence set into a structured gate package for expert review in G27. | semantic_gist (locked Situation, Pain, Benefit, AS set, traceability report), retrieved_artifacts (cited set from G19), relational_map, focal_entities, goal_orientation, uncertainty_signal | semantic_gist (gate package: structured summary of all locked outputs), retrieved_artifacts (final cited set), goal_orientation, predictive_cue (ready for G27) | Gate package must include: locked Situation, Pain, Benefit, AS set, traceability report, evidence citations, and any residual uncertainty_signals. |
| G27 | Gate Presentation | [E] | Present the gate package to the expert and obtain an explicit decision to proceed to the Approach phase or redirect for revision. | semantic_gist (gate package from G26), retrieved_artifacts (final cited set), focal_entities, goal_orientation, uncertainty_signal (residual), predictive_cue (G26 readiness signal) | episodic_trace (expert decision), semantic_gist (gate decision: PROCEED or REDIRECT with rationale), goal_orientation, predictive_cue (PROCEED → Approach phase; REDIRECT → re-entry step per G-REDIRECT) | Only step where inter-phase transition is authorized. /ty proceeds to Approach; /gm activates G-REDIRECT with gate package as context. |
| G-REDIRECT | /gm Handler | [conditional] | Update CCS to reflect the redirect instruction and identify the correct re-entry step, invalidating all CCS components whose source steps fall at or after that re-entry point. | episodic_trace (/gm instruction), semantic_gist (current state at redirect), focal_entities, goal_orientation, uncertainty_signal | semantic_gist (redirect rationale, re-entry step ID), goal_orientation, predictive_cue (re-entry step ID), uncertainty_signal (updated with redirect reason and stale component list) | **CCS invalidation rule:** On re-entry to step X, all components populated by steps ≥ X are stale; enumerate them explicitly in CCS Out. **Fork-join rule:** If re-entry target ≤ G5a, re-run both G5a and G5b and G5c. |

|---------|--------|

| H3 | G20 now produces predictive_cue (re-entry step ID) whenever an inconsistency is found. Re-entry rules: Situation–Pain mismatch → G3; Pain–Benefit mismatch → G15; multi-element → earliest inconsistent element. |
| H4 | Swapped G14 and G15. Similar Cases Research (G14, [R]) now precedes Elicitation Scoping — Benefit (G15, [I]) so that Benefit questions are grounded in real analogues. |
| H5 | Added G27 [E] Gate Presentation. Only step where inter-phase transition is authorized. Previously the Output Gate had no expert-facing step in the list. |
| M1 | Added G17b [I] Benefit Quality Validation between G17 and G18. Independently checks (a) not inverse of Pain, (b) traces to G13 root cause. BLOCK routes back to G17 with specific failure detail. |
| M2 | Renamed G5b to "Situation Domain Quantitative Research." Added explicit scope constraint: prohibited from pre-selecting Pain candidates. |
| M3 | Added G6b [E] Situation Validation Review after G6. Presents G6's validation findings; pass-through if no flag. |
| M4 | Added circuit-breaker notes to G4, G9, G17, G22: max N=5, escalation Turn on limit reached. |
| M5 | G-REDIRECT now enumerates stale CCS components on re-entry and includes fork-join rule. |
| M6 | goal_orientation added to CCS In for all [E] steps: G1, G3, G4, G6b, G8, G9, G10, G16, G17, G18, G21, G22, G27. |
| M7 | G24 failure-path CCS Out specified separately from success-path. G22 CCS In includes G23 uncertainty_signal on re-entry. |
| L1 | G11 language: "incorporating prior Situation artifacts" instead of "appended." |
| L2 | G21 AS format definition added: observable user-visible state; no solution mechanism; no internal system state. G23 references this. |

| L4 | G-REDIRECT fork-join rule: re-entry ≤ G5a requires re-running G5a, G5b, and G5c. |
| L5 | G19 citation definition: cited = referenced in Lock step CCS Out or locked semantic_gist. Uncited artifacts dropped. |
