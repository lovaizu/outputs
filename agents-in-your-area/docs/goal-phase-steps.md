# Goal Phase — Step List v5

Produced by integrating two independent expert reviews (product discovery + AI agent design)
and a subsequent re-review pass, then refined against Action Principles assay findings.

---

## Conventions

**Step types**

| Code | Meaning |
|------|---------|
| [I] | AI-internal step — no user interaction; runs to completion and writes CCS |
| [E] | Expert gate — requires `/ty` or `/gm` before any next step executes |
| [R] | Research step — reads external sources and writes findings to CCS; no state changes to goal artifacts |
| [conditional] | Executes only when a specified trigger condition is met; skip path is explicitly defined |

**Commands:** `/ty` — approve and proceed. `/gm` — provide feedback or redirect; opens a revision loop or activates G-REDIRECT.

**CCS** (Compressed Cognitive State) — the bounded state passed between Turns. Nine components: `episodic_trace` (what just happened), `semantic_gist` (current task framing), `focal_entities` (entities being worked on), `relational_map` (how entities relate), `goal_orientation` (the end goal — Benefit + AS, unchanged once locked), `constraints` (what must not be done), `predictive_cue` (what to do next), `uncertainty_signal` (what is still uncertain), `retrieved_artifacts` (evidence with source citations). These definitions make this artifact self-contained; [acc.md](acc.md) documents implementation details (component encoding, session initialization).

**Phase completion state:** Goal phase is complete when: (1) `goal.md` is committed to the branch — verified by G26a: file exists at expected path with all required sections; (2) Situation, Pain, Benefit, and AS are all locked — verified by CCS lock flags (Situation: G6/G6b resolution; Pain: G12; Benefit: G19; AS: G24); (3) traceability chain (AS→Pain→Situation) has no broken links — verified by G25 traceability report showing zero orphans; (4) G27 `/ty` has been issued by the expert — verified by G27 CCS Out episodic_trace recording the /ty decision.
