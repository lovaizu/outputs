# Approach Phase — Step List v9

Produced by integrating two independent expert reviews (product discovery + AI agent design)
and a subsequent re-review pass, then refined against Action Principles assay findings (iterations 1–6).

---

## Conventions

**Step types**

| Code | Meaning |
|------|---------|
| [I] | AI-internal step — no user interaction; runs to completion and writes CCS |
| [E] | Expert gate — requires `/ty` or `/gm` before any next step executes |
| [R] | Research step — reads sources (codebase or ecosystem) and writes findings to CCS; no state changes to approach artifacts |
| [conditional] | Executes only when a specified trigger condition is met; skip path is explicitly defined |

**Commands:** `/ty` — approve and proceed. `/gm` — provide feedback or redirect; opens a revision loop or activates A-REDIRECT.

**CCS** (Compressed Cognitive State) — the bounded state passed between Turns. Nine components: `episodic_trace` (what just happened), `semantic_gist` (current task framing), `focal_entities` (entities being worked on), `relational_map` (how entities relate), `goal_orientation` (the end goal — Benefit + AS verbatim from goal.md, unchanged throughout Approach), `constraints` (what must not be done), `predictive_cue` (what to do next), `uncertainty_signal` (what is still uncertain), `retrieved_artifacts` (evidence with source citations). These definitions make this artifact self-contained; [acc.md](acc.md) documents implementation details (component encoding, session initialization). **AI-internal use:** CCS In/Out columns document AI state management only; experts see only the content described in each [E] step's Presentation structure notes. CCS component names are not exposed in expert-facing output.

**Phase completion state:** Approach phase is complete when: (1) `approach.md` is committed to the branch — verified by A18a: file exists at expected path with all required sections; (2) Testing, Technology, and Design are all locked — verified by CCS lock flags (Testing: A6b /ty; Technology: A12; Design: A16); (3) AS-coverage matrix has no orphaned AS rows — verified by A17 consistency report showing zero broken links; (4) all NOFIT compromise rows have been explicitly acknowledged by the expert — verified by A19 CCS In showing each NOFIT row acknowledged; (5) A19 `/ty` has been issued — verified by A19 CCS Out episodic_trace recording the /ty decision.

**Locked element summary:**

| Element | Locked at | What locking means |
|---------|-----------|-------------------|
| Testing | A6 (confirmed A6b) | Test runner, patterns, and AS→method mapping committed; all downstream steps treat Testing as fixed |
| Technology | A12 | Technology choice committed; must/must_not constraints derived; Design may begin |
| Design | A16 | Design decisions committed; A17 consistency validation may proceed |
